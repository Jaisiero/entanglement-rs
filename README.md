# entanglement-rs

Rust bindings and high-level async API for the Entanglement C++ networking library. Provides safe, tokio-integrated client and server abstractions over the low-level UDP transport.

## Overview

This is a Cargo workspace with two crates:

- **`entanglement-sys`** — Raw FFI bindings generated by `bindgen` from the C++ Entanglement library
- **`entanglement-rs`** — High-level async Rust API built on top of `entanglement-sys`

## Workspace Structure

```
entanglement-rs/
├── entanglement-sys/       Raw C++ bindings
│   ├── build.rs            Compiles C++ sources + runs bindgen
│   ├── deps/Entanglement/  C++ source (git submodule)
│   └── src/lib.rs          Generated bindings re-export
├── entanglement-rs/        High-level Rust API
│   └── src/
│       ├── server.rs       Async server: event loop, sessions, endpoints
│       ├── client.rs       Async client: connect, send, receive
│       ├── error.rs        Error types from C error codes
│       └── lib.rs          Module exports
└── Cargo.toml              Workspace root
```

## API

### EntClient

```rust
let client = EntClient::new(config)?;
client.connect("192.168.1.30:9876").await?;
client.send(&payload).await?;
let data = client.recv().await?;
```

### EntServer

Async server managing client sessions with Tokio channels for event-driven message handling.

## Build

```bash
cargo build --release
```

### Build Requirements

- C++20 compiler (GCC 10+ / Clang 12+)
- On Windows: links `ws2_32` (Winsock)
- On Linux: links `libbpf`, `libxdp` (for AF_XDP zero-copy support)

The build compiles all `.cpp` files from `deps/Entanglement/src/` and generates Rust bindings from `deps/Entanglement/include/entanglement.h`.
